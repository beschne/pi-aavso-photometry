1 Introduction

[hide]

This script performs differential PSF photometry of a variable star against one or more comparison stars of known magnitude, using only the tools already present in PixInsight: the WCS astrometric solution from ImageSolver, the Gaussian PSF fitter from DynamicPSF, and the FITS keywords written by the standard calibration and integration pipeline.

The script is currently preconfigured for T Coronae Borealis (T CrB), a recurrent nova known as the “Blaze Star”. At quiescence T CrB sits near 10th magnitude; an outburst is predicted to bring it to approximately 2nd magnitude — an increase of roughly 8 magnitudes in a few days. The script measures the brightness of T CrB in the TG (tri-colour green) photometric band and writes the result in the AAVSO Extended File Format for direct upload to AAVSO Submit Photometric Observations.

The script works on a single linear, plate-solved OSC RGB master light stack. It measures each star by fitting a two-dimensional Gaussian PSF to the green channel, computes instrumental magnitudes from the fitted flux, and applies the standard differential photometry formula:

mag(T CrB) = mag_catalogue(comp) + instMag(T CrB) − instMag(comp)

A check star provides an independent quality gate: if its derived magnitude deviates from catalogue by more than 3×MERR, a warning is shown.

2 Requirements

[hide]

PixInsight 1.9.4 “Lockhart” or later

The script targets the V8 JavaScript engine introduced in PixInsight 1.9.4. Earlier releases used SpiderMonkey 24 and are not supported.

Linear, plate-solved OSC RGB master light stack

The active image must satisfy the following conditions:

  • Linear (unstretched) — PSF flux is non-linear on a stretched image; photometry on a stretched image is meaningless.
  • Plate-solved — an astrometric WCS solution must be stored in the image (run ImageSolver if needed).
  • RGB (three channels) — the script uses the green channel (index 1).

The following processes are compatible because they cancel in the differential magnitude calculation: background extraction (ABE, DBE, GradientCorrection) and SpectrophotometricColorCalibration (SPCC).

The following processes are incompatible and will be flagged by the script if their names appear in the FITS HISTORY keywords: HistogramTransformation, GHSStretch, MaskedStretch, CurveTransformation, Deconvolution, BlurXTerminator, LocalHistogramEqualization, ExponentialTransformation, HDRMultiscaleTransform.

AAVSO VSP comparison-star CSV

A photometry table exported from the AAVSO Variable Star Plotter in CSV format. The script filters for V-band rows and projects each in-frame star onto the image using the plate-solve solution.

3 Getting Started

[hide]

3.1 Download the Comparison-Star CSV

  1. Go to the AAVSO Variable Star Plotter: aavso.org/vsp
  2. Enter target name: T CrB
  3. Set Field of view to 60′ (matches a typical small-scope or Seestar field)
  4. Set Limiting magnitude to 12.0
  5. Click Plot Chart and note the Chart ID (e.g. X42597QE)
  6. Click Photometry table → Download → save as CSV

A reference copy for chart X42597QE is distributed with this script in docs/X42597QE_photometry.csv.

3.2 Prepare the Image

Open your linear, plate-solved OSC RGB master stack in PixInsight as the active window. If the stack has not yet been plate-solved, run Script > Astronomy > ImageSolver first.

The script reads FITS HISTORY keywords to detect incompatible processes. These keywords are only written when the master is saved to disk with the “Add FITS keywords” preference enabled. If you applied any stretch or sharpening in-session without saving, the detection will have a blind spot; the green-channel median heuristic on the Photometry step provides a supplementary linearity check in that case.

3.3 Run the Script

  • Quick run (no menu entry): Script > Execute Script File… → select aavso-photometry.js. The script is re-read from disk on every run; no restart is required after editing.
  • With menu entry: Script > Feature Scripts… > Add the script’s directory once. PixInsight registers it under Script > BeSchne > Photometry. Code changes are picked up automatically on the next run; only changes to the #feature-id registration line require a rescan or relaunch.

4 Dialog Reference

[hide]

The script presents a six-step wizard. A narrow left pane lists the steps; clicking a step label navigates to it. The Next › button at the bottom right advances to the next step. Photometry (step 3) and the Report (step 6) are generated automatically on entry; they do not require a separate button click.

4.1 Step 1 — Setup

Active image

Read-only display of the currently active PixInsight image window. The script operates on this image. If no image is open, or the active image is not a plate-solved RGB stack, the script aborts with a descriptive error message.

Comparison CSV

Path to the AAVSO VSP photometry CSV downloaded in section 3.1. The path is remembered between sessions; the script re-prompts if the file no longer exists. Click Browse… to select a different file. The script reads all V-band rows, projects each star onto the image, and discards any that fall outside the frame or are flagged as blended in the CSV comments.

Observer code

Your AAVSO observer code, written into the #OBSCODE= line of the report header. The default is BSLA; change it to your own code before exporting. The value is remembered between sessions.

4.2 Step 2 — Comp Stars

This step runs a PSF discovery pass on all in-frame V-band candidates from the CSV and presents them in a table. Stars that pass quality checks and are within 2 magnitudes of the target are pre-selected as the ensemble. The discovery pass is shared with the photometry step; no second PSF fit is performed.

Star table

Each row shows the star’s AAVSO chart label, AUID, catalogue V magnitude, Δmag from the target, and PSF quality notes. A check mark in the first column indicates the star is included in the ensemble. Click any row to toggle inclusion. Stars outside the image frame, flagged as blended, or failing PSF quality checks are shown greyed out and cannot be selected.

Check star

Selected from the drop-down at the bottom of the panel. The check star is kept separate from the ensemble; it is not used to compute the zero-point. Its derived magnitude is compared to catalogue as an independent quality gate; a deviation greater than 3×MERR triggers an orange warning on the Photometry step.

4.3 Step 3 — Photometry

Photometry runs automatically when this step is entered. The panel shows the results immediately.

Magnitude / Filter / Error

The derived magnitude of T CrB in the TG band, the filter label (TG), and the magnitude error MERR. MERR is propagated from the PSF fit residuals of both the target and the comparison star, combined in quadrature using a matched-filter formula.

Raw PSF flux (instrumental magnitudes)

The instrumental magnitudes −2.5 log10(flux) for the target (T CrB), the ensemble (mean zero-point from all selected comp stars), and the check star, with labels. These are the raw values before the catalogue offset is applied. Negative instrumental magnitudes are normal for bright stars.

Ensemble / Check star

The number of ensemble comp stars (N) and their combined zero-point, plus the AAVSO chart label and catalogue V magnitude of the check star.

Linearity warning (yellow / orange)

A supplementary heuristic checks the green-channel median of the active image. A linear master typically has a median below 0.05 (normalised). Values between 0.05 and 0.15 trigger a yellow advisory; values above 0.15 trigger an orange warning suggesting the image may be stretched. This is a heuristic only and does not block the report.

Forbidden-process warning (red)

If a FITS HISTORY keyword contains the name of an incompatible process (stretch, deconvolution, BlurXTerminator, …), a red label lists the offending process names. This check is limited to processes recorded in the FITS keywords; see Requirements for the known blind spot.

Check-star gate (orange)

If the check star’s derived magnitude deviates from its catalogue V by more than 3×MERR, an orange label reports the deviation. This suggests a systematic problem such as a wrong star, blending, or poor atmospheric conditions. The report can still be exported, but the observation should be scrutinised before submission.

4.4 Step 4 — Mid-time

This step establishes the observation timestamp written as DATE (mid-exposure JD) in the AAVSO report and used to compute airmass.

Start / End (reference-file buttons)

Click the folder buttons to select FITS or XISF subframe files. The script reads DATE-OBS and EXPTIME from each file’s header. Select the first surviving subframe for Start and the last surviving subframe for End. End is computed as the last sub’s DATE-OBS + EXPTIME. Both fields are also directly editable in UTC ISO-8601 format.

Note: Do not rely on the master stack’s own DATE-OBS keyword for the session span. PixInsight’s ImageIntegration writes the midpoint of sub start times, ignoring exposure duration, which gives a systematically early result.

Mid-time

Three modes, selectable by radio button:

  • (Start+End)/2 (default, recommended) — arithmetic midpoint of the session span.
  • = Start — use the start of the first subframe. Appropriate for very short or single-frame sessions.
  • Manual — type an arbitrary UTC ISO-8601 time into the edit field.

The resulting mid-time JD and its ISO-8601 equivalent are shown as read-only readouts.

Lat / Lon / Elev

Observer site coordinates, pre-filled from FITS keywords (SITELAT, SITELONG, SITEELEV and synonyms). All three fields are editable. Longitude is degrees East-positive. Elevation defaults to 0 m if absent. If Lat or Lon remain blank, the report writes AMASS=na, which is valid per the AAVSO Extended File Format specification.

Airmass

Computed using the Kasten & Young (1989) formula from the confirmed mid-time JD, the observer site coordinates, and T CrB’s J2000 position. Shown as a read-only readout; written as AMASS in the report.

Moon

Approximate lunar phase (percentage illumination) and angular separation from T CrB at the mid-time, shown as a sanity check. Not written to the report.

4.5 Step 5 — Verification

This step displays an annotated thumbnail of the image centred on T CrB. Confirm that the coloured circles land on the correct stars before exporting the report.

Annotation circles

  • Red — T CrB (the target)
  • Green — ensemble comparison stars
  • Cyan — check star (Check)

Stretch

Three display modes for the thumbnail:

  • No stretch — raw linear values; very faint appearance for a typical deep-sky master.
  • Auto stretch — midtone transfer function matched to the image median; stars are clearly visible.
  • Boosted — aggressive stretch that brings out faint stars; useful for confirming the check star position.

Changing the stretch re-renders the thumbnail without leaving the dialog. The stretch applied here does not affect the photometry; measurements were already computed in step 3.

4.6 Step 6 — Report

The report is generated automatically when this step is entered.

Human-readable format (default)

A formatted text summary of the observation: target, magnitude, filter, time, airmass, comp/check details, and a brief quality summary. Suitable for personal records or quick review.

AAVSO Extended Format

A 15-field comma-delimited CSV observation line preceded by the required AAVSO file header. Submit via the AAVSO Submit Photometric Observations form: select File upload in the first field, upload the .txt file, review the preview plot, and click Submit. See section 6 for field details. This workflow has been tested end-to-end with a T CrB ensemble submission.

Export…

Opens a save-file dialog. Save with a .txt extension. The file is written immediately; the dialog preview is a live view of what will be saved.

5 Outburst Strategy

[hide]

T CrB is expected to brighten by approximately 8 magnitudes from quiescence (~10 V) to nova peak (~2 V). As the target brightens the optimal comparison stars change.

Early brightening (~7–9 mag): Chart X42597QE contains brighter comparison stars. In the Comp Stars step (step 2), untick the fainter stars and tick the brighter ones; also update the Check drop-down accordingly:

LabelAUIDV magUse when T CrB is near
98000-BBW-7969.809Quiescence (~10 V)
106000-BJS-90110.554Quiescence (check star)
84000-BBW-8888.361Brightening, ~7–9 mag
79000-BBW-8817.886Brightening, ~7–9 mag

Nova peak (~2–6 mag): The comparison stars in X42597QE will be too faint relative to T CrB, and the star itself will saturate normal exposures. A new AAVSO VSP chart with a wider field (180′) and brighter magnitude limit (5–6) is required. AAVSO issues Alert Notices with specific chart and exposure guidance when the outburst begins; monitor aavso.org/news.

6 Output Format

[hide]

The AAVSO Extended File Format report consists of a file header followed by one observation line.

File header:

#TYPE=EXTENDED
#OBSCODE=<observer code>
#SOFTWARE=AAVSO Photometry 1.2.0
#DELIM=,
#DATE=JD
#OBSTYPE=CCD

Observation fields (15 comma-delimited fields):

NAME

Target identifier: T CRB.

DATE

Mid-exposure Julian Date (JD), as confirmed in step 3.

MAG

Derived TG magnitude of T CrB.

MERR

Magnitude error, propagated from PSF fit residuals (MAD) of the target and comparison star combined in quadrature.

FILT

TG (tri-colour green, OSC green channel). Never V — TG and Johnson V differ by 0.1–0.3 mag for red stars.

TRANS

NO — no colour transformation applied.

MTYPE

STD — standard (not differential) magnitude output.

CNAME

When N = 1: AUID of the single comparison star (e.g. 000-BBW-796). When N > 1: ENSEMBLE. The list of ensemble star labels appears in NOTES.

CMAG

When N = 1: instrumental magnitude of the comparison star (can be negative). When N > 1: na. Per AAVSO spec for MTYPE=STD with ensemble.

KNAME

AUID of the check star.

KMAG

Instrumental magnitude of the check star.

AMASS

Airmass at mid-time (Kasten & Young 1989). Written as na if site latitude or longitude is absent.

GROUP

na.

CHART

AAVSO VSP chart ID (e.g. X42597QE).

NOTES

Script name and version; for ensemble observations, also lists the comp star labels used (e.g. comp 98+106+93;).

7 Known Limitations

[hide]

TG band ≠ Johnson V

The script measures in the TG band (OSC green channel), not Johnson V. For red stars, TG runs ~0.1–0.3 mag brighter than V because the OSC green passband has a different effective wavelength than the standard V filter. T CrB has an M3 III companion that dominates the combined light at quiescence, making this offset significant. The AAVSO report correctly records FILT=TG and TRANS=NO. Never relabel TG measurements as V.

Single-comp fallback

Selecting only one star in the Comp Stars step is mathematically equivalent to single-comparison-star differential photometry. The default is to use all qualifying in-frame V-band stars as an ensemble, which averages out individual comparison-star errors.

DATE-OBS unreliable in stacked masters

PixInsight’s ImageIntegration writes DATE-OBS as the midpoint of sub start times, ignoring exposure duration. This gives a systematically early result. Always use the reference-file buttons in the Mid-time step to set Start and End from actual subframes.

In-memory stretch detection blind spot

FITS HISTORY keywords are only written when a file is saved to disk. A stretch or deconvolution applied to an in-memory ImageIntegration result in the current session, without an intermediate save, will not appear in the keywords and will not be detected. The green-channel median heuristic on the Photometry step provides a supplementary check.

FOV constraint at nova peak

At maximum brightness T CrB will saturate any normal exposure, and the faint comparison stars in chart X42597QE may be undetectable. A wider-field instrument and a new AAVSO VSP chart with brighter comp stars will be required. See section 5.
